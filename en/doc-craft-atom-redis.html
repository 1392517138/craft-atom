Introduction

A easy use redis java client base on jedis.

What it provides and doesn't provides?


1.  redis commands
    Now it is fully compatible with all commands supported by redis 2.6.17.
2.  redis transactions
    A simple and elegant batch execution mechanism.
3.  redis pub/sub
    A simple publish/subscribe messaging paradigm.
4.  redis sharding
    Default murmur hash strategy, and support extended hash strategy implementor.
5.  redis master slave
    An easy way to interact with (sharding) master slave redis architect.
6.  redis pipelining
    Make a very difficult decision to give up redis pipelinling.

Getting Started

By default we support four type redis deployment architect.

Type 1 - singleton redis
Just a singleton redis server instance, figure 1 is the deployment architect diagram. A typical usage idiom for this type as follows: 


Redis redis = RedisFactory.newRedisBuilder(host, port).timeoutInMillis(1000).build();

Type 2 - sharded redis
Sharded multiple redis server instances, figure 2 is the deployment architect diagram. A typical usage idiom for this type as follows: 


// shardstring format string e.g. localhost:6379,localhost:6380,localhost:6381
ShardedRedis shardedRedis = RedisFactory.newShardedRedisBuilder(shardstring).timeoutInMillis(1000).build();

Type 3 - master slave redis
Master slave redis server instances, one master and multiple slaves, figure 3 is the deployment architect diagram. A typical usage idiom for this type as follows: 


// masterslavestring format string e.g. localhost:6379-localhost:6380-localhost:6381 the first is master, others are slaves.
MasterSlaveRedis masterSlaveRedis = RedisFactory.newMasterSlaveRedisBuilder(masterslavestring).timeoutInMillis(1000).build();

Type 4 - master slave sharded redis
Master slave sharded redis server instances, multiple sharded redis instance and each shard has one or more slaves , figure 4 is the deployment architect diagram. A typical usage idiom for this type as follows: 


// masterslaveshardstring format string e.g. localhost:6379-localhost:6380,localhost:6389-localhost:6390
MasterSlaveShardedRedis masterSlaveShardedRedis = RedisFactory.newMasterSlaveShardedRedisBuilder(masterslaveshardstring).timeoutInMillis(1000).build();
